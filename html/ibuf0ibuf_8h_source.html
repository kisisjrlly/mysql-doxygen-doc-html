77 /* The purpose of the insert buffer is to reduce random disk access.
78 When we wish to insert a record into a non-unique secondary index and
79 the B-tree leaf page where the record belongs to is not in the buffer
80 pool, we insert the record into the insert buffer B-tree, indexed by
81 (space_id, page_no). When the page is eventually read into the buffer
82 pool, we look up the insert buffer B-tree for any modifications to the
83 page, and apply these upon the completion of the read operation. This
84 is called the insert buffer merge. */
85 
86 /* The insert buffer merge must always succeed. To guarantee this,
87 the insert buffer subsystem keeps track of the free space in pages for
88 which it can buffer operations. Two bits per page in the insert
89 buffer bitmap indicate the available space in coarse increments. The
90 free bits in the insert buffer bitmap must never exceed the free space
91 on a page. It is safe to decrement or reset the bits in the bitmap in
92 a mini-transaction that is committed before the mini-transaction that
93 affects the free space. It is unsafe to increment the bits in a
94 separately committed mini-transaction, because in crash recovery, the
95 free bits could momentarily be set too high. */
96

129 /** Updates the free bits of an uncompressed page in the ibuf bitmap if there
130 is not enough free on the page any more. This is done in a separate
131 mini-transaction, hence this operation does not restrict further work to only
132 ibuf bitmap operations, which would result if the latch to the bitmap page were
133 kept. NOTE: The free bits in the insert buffer bitmap must never exceed the
134 free space on a page. It is unsafe to increment the bits in a separately
135 committed mini-transaction, because in crash recovery, the free bits could
136 momentarily be set too high. It is only safe to use this function for
137 decrementing the free bits. Should more free space become available, we must
138 not update the free bits here, because that would break crash recovery.
139 @param[in] block index page to which we have added new records;
140  the free bits are updated if the index is
141  non-clustered and non-unique and the page level
142  is 0, and the page becomes fuller
143 @param[in] max_ins_size value of maximum insert size with reorganize
144  before the latest operation performed to the
145  page
146 @param[in] increase upper limit for the additional space used in
147  the latest operation, if known, or
148  ULINT_UNDEFINED */
