344 /**
345  * Initialize logfile handler
346  *
347  * Initializes handler which will handle application's log. This handler
348  * will be attached to all currently-registered loggers.
349  * If `logging_folder` is provided, handler will log messages to logfile; its
350  * path and filename will be derived from `program` and `logging_folder`
351  * parameters.
352  * If `logging_folder` is empty, handler will log messages to console, unless
353  * `use_os_log` is set to true, in which case it will log to system logger
354  * instead (i.e. Syslog, Windows Eventlog, etc. Currently, only Windows
355  * Eventlog is supported).
356  *
357  * @param registry Registry object, typically managed by DIM
358  * @param program Name of the main program (Router)
359  * @param logging_folder logging_folder provided in configuration file
360  * @param format_messages If set to true, log messages will be formatted
361  * (prefixed with log level, timestamp, etc) before logging
362  * @param use_os_log If true, use system logger instead of STDERR (currently,
363  * only Windows Eventlog is supported)
364  *
365  * @throws std::runtime_error if opening log file or OS log fails
366  */
367 HARNESS_EXPORT
