227 /// The class represents parse-time context, which keeps track of declared
228 /// variables/parameters, conditions, handlers, cursors and labels.
229 ///
230 /// sp_context objects are organized in a tree according to the following
231 /// rules:
232 /// - one sp_pcontext object corresponds for for each BEGIN..END block;
233 /// - one sp_pcontext object corresponds for each exception handler;
234 /// - one additional sp_pcontext object is created to contain
235 /// Stored Program parameters.
236 ///
237 /// sp_pcontext objects are used both at parse-time and at runtime.
238 ///
239 /// During the parsing stage sp_pcontext objects are used:
240 /// - to look up defined names (e.g. declared variables and visible
241 /// labels);
242 /// - to check for duplicates;
243 /// - for error checking;
244 /// - to calculate offsets to be used at runtime.
245 ///
246 /// During the runtime phase, a tree of sp_pcontext objects is used:
247 /// - for error checking (e.g. to check correct number of parameters);
248 /// - to resolve SQL-handlers.
249 
250 class sp_pcontext {
251  public:
252  enum enum_scope {
253  /// REGULAR_SCOPE designates regular BEGIN ... END blocks.
254  REGULAR_SCOPE,
255 
256  /// HANDLER_SCOPE designates SQL-handler blocks.
257  HANDLER_SCOPE
258  };
