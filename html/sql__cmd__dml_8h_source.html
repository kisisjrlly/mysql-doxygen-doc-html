117  /**
118  Perform a precheck of table privileges for the specific operation.
119 
120  @details
121  Check that user has some relevant privileges for all tables involved in
122  the statement, e.g. SELECT privileges for tables selected from, INSERT
123  privileges for tables inserted into, etc. This function will also populate
124  TABLE_LIST::grant with all privileges the user has for each table, which
125  is later used during checking of column privileges.
126  Note that at preparation time, views are not expanded yet. Privilege
127  checking is thus rudimentary and must be complemented with later calls to
128  SELECT_LEX::check_view_privileges().
129  The reason to call this function at such an early stage is to be able to
130  quickly reject statements for which the user obviously has insufficient
131  privileges.
132  This function is called before preparing the statement.
133  The function must also be complemented with proper privilege checks for all
134  involved columns (e.g. check_column_grant_*).
135  @see also the function comment of SELECT_LEX::prepare().
136  During execution of a prepared statement, call check_privileges() instead.
137 
138  @param thd thread handler
139 
140  @returns false if success, true if false
141  */
142  virtual bool precheck(THD *thd) = 0;
