26 /**
27  Session_sysvar_resource_manager
28  -------------------------------
29  When a session (THD) gets initialized, it receives a shallow copy of all
30  global system variables.
31  thd->variables= global_system_variables; (see plugin_thdvar_init())
32 
33  In case of Sys_var_charptr variables, we need to maintain a separate copy for
34  each session though so that global and session variables can be altered
35  independently.
36 
37  This class is responsible for alloc|dealloc-ating memory for Sys_var_charptr
38  variables for every session. It works in three steps :
39 
40  (1) init :
41  Creates a copy (memdup()) of global Sys_var_charptr system variable for
42  the respective session variable (passed as a parameter) & inserts it
43  into sysvar_string_alloc_hash (containing the alloced address) to infer
44  that memory has been allocated for the session. init() is called during
45  the initialization of session system variables. (plugin_thdvar_init())
46  (2) update :
47  When the session variable is updated, the old memory is freed and new
48  memory is allocated to hold the new value. The corresponding member in
49  sysvar_string_alloc_hash is also updated to hold the new alloced memory
50  address. (Sys_var_charptr::session_update())
51  (3) deinit :
52  Its a one-shot operation to free all the session Sys_var_charptr system
53  variables. It basically traverses down the sysvar_string_alloc_hash
54  hash and calls free() for all the addresses that it holds.
55 
56  Note, there should always be at most one node per Sys_var_charptr session
57  system variable.
58 
59 */
