84 /** @class CmdArgHandler
85  * @brief Handles command line arguments
86  *
87  * The CmdArgHandler class handles command line arguments. It is a
88  * replacement and supports most of the POSIX GNU getopt library.
89  *
90  * Command line options can have multiple aliases. For example, the
91  * typical `--help` can be also called `-h`, or even `--help-me`. Long
92  * names starting with one dash are not supported.
93  *
94  * Command line options are added through the `add_option()` method
95  * and can be given 1 or more names and a description. It is also
96  * possible to require the option to have a value, or make the value
97  * optional.
98  *
99  * During processing of the command line arguments, actions will be
100  * executed when the option (and it's potential value) was found.
101  *
102  * Usage example:
103  *
104  * #include "arg_handler.h"
105  *
106  * void MyApp::prepare_command_line_options() {
107  * // CmdArgHandler handler_;
108  * handler_.add_option(OptionNames({"-h", "--help", "--sos"}), "Show help
109  * screen", CmdOptionValueReq::none, "", [this](const string &) {
110  * this->show_help(); } handler_.add_option(OptionNames({"--config"}),
111  * "Configuration file", CmdOptionValueReq::none, "", [this](const string
112  * &value) { this->set_config_file(value); },
113  * []{});
114  *
115  * void MyApp::init(const vector<string> arguments) {
116  * prepare_command_line_options();
117  * handler_.process(arguments);
118  * }
119  *
120  * All arguments which are not valid option names are are not values of options
121  * are considered rest arguments. By default, rest arguments are not allowed and
122  * will raise a std::invalid_argument exception. It is possible to allow them
123  * through the constructor.
124  *
125  * The CmdArgHandler class also provides functionality to help creating help
126  * screen or show how to use the the command line application. The method
127  * `usage_lines()` produces a usage line showing all the option names, their
128  * required or optional value with a meta variable. Similar, method
129  * `option_descriptions()` will get all options and their descriptions. All this
130  * is text wrapped at a configurable margin as well as, if needed, indented.
131  *
132  * @internal
133  * The command line argument handling in CmdArgHandler is the bare minimum
134  * needed for MySQL Router. It was needed to make sure that the application
135  * would compile on system where the getopt library is not available.
136  * @endinternal
137  *
138  */

154  /** @brief Adds a command line option
155  *
156  * Adds a command line option given names, description, value
157  * requirement and optional action.
158  *
159  * `names` is a vector of strings which contains names starting
160  * with either a single or double dash, `-` or `--`. It is possible
161  * to add more than one name for an option.
162  *
163  * The description text will be used in the help output. Note that
164  * this can be a very long as text will be wrapped (and optionally
165  * indented). New lines in the description will be respected.
166  *
167  * The `metavar` argument is used in the usage text as a placeholder
168  * for the (optional) value of the option, for example, when `metavar`
169  * is set to `path`, the usage would show:
170  *
171  * --config=<path>
172  *
173  * The value_req argument should be either:
174  *
175  * * `CmdOptionValueReq::none` : option has no value
176  * * `CmdOptionValueReq::required` : option requires value
177  * * `CmdOptionValueReq::optional` : option has optional value
178  *
179  * The `action` argument should be a `std::function` and is called
180  * with the (optional) value of the option. The function should
181  * accept only a `const std::string`.
182  *
183  * The `at_end_action` argument should be a `std::function`. This is optional
184  * argument, if not provided then []{} is used as at_end_action. The
185  * `at_end_action` is ment to be used for additional validation, if particular
186  * set of options has to be used together, or if particular set of options
187  * cannot be used together.
188  *
189  * Example usage:
190  *
191  * handler_.add_option(OptionNames({"--config"}), "Configuration file",
192  * CmdOptionValueReq::none, "",
193  * [this](const string &value) {
194  * this->set_config_file(value); },
195  * []{});
196  *
197  * @internal
198  * The `add_option` method will assert when `names` is empty,
199  * one of the names is not valid or when a name was already used with
200  * another option.
201  * @endinternal
202  *
203  * @param names vector of string with option names, each starting with - or --
204  * @param description descriptive text explaining the option
205  * @param value_req value requirement of the option
206  * @param metavar for formatting help text when option accepts a value
207  * @param action action to perform when the option was found
208  * @param at_end_action task to perform after all actions have been done
209  */
210  void add_option(
211  const CmdOption::OptionNames &names, const std::string &description,
212  const CmdOptionValueReq &value_req, const std::string &metavar,
213  CmdOption::ActionFunc action,
214  CmdOption::AtEndActionFunc at_end_action = [](const std::string &) {
215  }) noexcept;

244  /** @brief Checks whether given name is a valid option name
245  *
246  * Checks whether the given name is a valid option name.
247  *
248  * A valid option name should:
249  *
250  * * have at least consist of 2 characters
251  * * start with a dash '-'
252  * * match the reqular expression ^--[A-Za-z]{2}[A-Za-z_-]+$
253  *
254  * It is allowed to use the equal sign when giving value. Following options
255  * are equal:
256  * --config /path/to/mysqlrouter.conf
257  * --config=/path/to/mysqlrouter.conf

259  * Throws std::invalid_argument when option name is not valid or
260  * option was not registered.
261  *
262  * Examples of valid option names:
263  *
264  * -h
265  * --with-ham
266  * --with_spam
267  *
268  * Example of invalid option names:
269  *
270  * -help
271  * ---spam
272  * --x-ham
273  *
274  * @param name option name to check

297  /** @brief Produces lines of text suitable to show usage
298  *
299  * Produces lines of text suitable to show usage of the command line
300  * appliation. Each option is shown with all its names and with optional
301  * or required value.
302  *
303  * The `prefix` argument can be used to add text, usually the name
304  * of the command, in front of the options. The lines are indented
305  * using the size of the prefix.
306  *
307  * The `rest_metavar` can be used to name non-options arguments.
308  *
309  * The `width` argument is used to set the maximum length of the lines.
310  *
311  * Example output when all lines are printed:
312  *
313  * usage: mysqlrouter [-v|--version] [-h|--help] [-c|--config=<path>]
314  * [-a=[<foo>]] [rest..]
315  *
316  *
317  * @param prefix text in front of options (usually command name)
318  * @param rest_metavar name of rest arguments (empty if not needed)
319  * @param width maximum length of each line
320  * @return vector of strings
321  */
322  std::vector<std::string> usage_lines(const std::string &prefix,
323  const std::string &rest_metavar,
