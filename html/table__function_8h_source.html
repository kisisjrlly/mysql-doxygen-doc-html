288  /**
289  Fill a json table column
290 
291  @details Fills a column with data, according to specification in
292  JSON_TABLE. This function handles all kinds of columns:
293  Ordinality) just saves the counter into the column's field
294  Path) extracts value, saves it to the column's field and handles
295  ON ERROR/ON EMPTY clauses
296  Exists) checks the path existence and saves either 1 or 0 into result
297  field
298  Nested path) matches the path expression against data source. If there're
299  matches, this function sets NESTED PATH's iterator over those
300  matches and resets ordinality counter.
301 
302  @param[in] table_function the JSON table function
303  @param[out] skip true <=> it's a NESTED PATH node and its path
304  expression didn't return any matches or a
305  previous sibling NESTED PATH clause still producing
306  records, thus all columns of this NESTED PATH node
307  should be skipped
308 
309  @returns
310  false column is filled
311  true an error occurred, execution should be stopped
312  */
313  bool fill_column(Table_function_json *table_function, jt_skip_reason *skip);
314 };
