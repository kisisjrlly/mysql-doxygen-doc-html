76 /**
77  @brief A facade to the functionality of the Event Scheduler.
78 
79  Every public operation against the scheduler has to be executed via the
80  interface provided by a static method of this class. No instance of this
81  class is ever created and it has no non-static data members.
82 
83  The life cycle of the Events module is the following:
84 
85  At server start up:
86  init_mutexes() -> init()
87  When the server is running:
88  create_event(), drop_event(), start_or_stop_event_scheduler(), etc
89  At shutdown:
90  deinit(), destroy_mutexes().
91 
92  The peculiar initialization and shutdown cycle is an adaptation to the
93  outside server startup/shutdown framework and mimics the rest of MySQL
94  subsystems (ACL, time zone tables, etc).
95 */
