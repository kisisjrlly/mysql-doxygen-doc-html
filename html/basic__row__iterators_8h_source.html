434 /**
435  FollowTailIterator is a special version of TableScanIterator that is used
436  as part of WITH RECURSIVE queries. It is designed to read from a temporary
437  table at the same time as MaterializeIterator writes to the same table,
438  picking up new records in the order they come in -- it follows the tail,
439  much like the UNIX tool “tail -f”.
440 
441  Furthermore, when materializing a recursive query expression consisting of
442  multiple query blocks, MaterializeIterator needs to run each block several
443  times until convergence. (For a single query block, one iteration suffices,
444  since the iterator sees new records as they come in.) Each such run, the
445  recursive references should see only rows that were added since the last
446  iteration, even though Init() is called anew. FollowTailIterator is thus
447  different from TableScanIterator in that subsequent calls to Init() do not
448  move the cursor back to the start.
449 
450  In addition, FollowTailIterator implements the WITH RECURSIVE iteration limit.
451  This is not specified in terms of Init() calls, since one run can encompass
452  many iterations. Instead, it keeps track of the number of records in the table
453  at the start of iteration, and when it has read all of those records, the next
454  iteration is deemed to have begun. If the iteration counter is above the
455  user-set limit, it raises an error to stop runaway queries with infinite
456  recursion.
457  */
458 class FollowTailIterator final : public TableRowIterator {
459  public:
460  // "examined_rows", if not nullptr, is incremented for each successful Read().
461  FollowTailIterator(THD *thd, TABLE *table, QEP_TAB *qep_tab,
462  ha_rows *examined_rows);

464 
465  bool Init() override;
466  int Read() override;
