50 /** @class MySQLRouter
51  * @brief Manage the MySQL Router application.
52  *
53  * The class MySQLRouter manages the MySQL Router application. It handles the
54  * command arguments, finds valid configuration files, and starts all plugins.
55  *
56  * Since MySQL Router requires at least 1 configuration file to be available
57  * for reading, if no default configuration file location could be read and no
58  * explicit location was given, the application exits.
59  *
60  * The class depends on MySQL Harness to, among other things, load the
61  * configuration and initalize all request plugins.
62  *
63  * Example usage:
64  *
65  * int main(int argc, char** argv) {
66  * MySQLRouter router(argc, argv);
67  * router.start();
68  * }
69  *
70  */

448  /** @brief Sets default configuration file locations
449  *
450  * Sets the default configuration file locations based on information
451  * found in the locations argument. The locations should be provided
452  * as a semicolon separated list.
453  *
454  * The previous loaded locations are first removed. If not new locations
455  * were provider (if locations argument is empty), then no configuration
456  * files will be available.
457  *
458  * Locations can include environment variable placeholders. These

463  *
