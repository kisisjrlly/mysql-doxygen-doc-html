313 /** Compute the next autoinc value.
314 
315  For MySQL replication the autoincrement values can be partitioned among
316  the nodes. The offset is the start or origin of the autoincrement value
317  for a particular node. For n nodes the increment will be n and the offset
318  will be in the interval [1, n]. The formula tries to allocate the next
319  value for a particular node.
320 
321  Note: This function is also called with increment set to the number of
322  values we want to reserve for multi-value inserts e.g.,
323 
324  INSERT INTO T VALUES(), (), ();
325 
326  innobase_next_autoinc() will be called with increment set to 3 where
327  autoinc_lock_mode != TRADITIONAL because we want to reserve 3 values for
328  the multi-value INSERT above.
329  @return the next value */
330 ulonglong innobase_next_autoinc(
331  ulonglong current, /*!< in: Current value */
332  ulonglong need, /*!< in: count of values needed */
333  ulonglong step, /*!< in: AUTOINC increment step */
334  ulonglong offset, /*!< in: AUTOINC offset */
335  ulonglong max_value) /*!< in: max value for type */
