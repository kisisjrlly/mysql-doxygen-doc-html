◆ allow_windows_service_to_write_logs()

void allow_windows_service_to_write_logs ( const std::string &  conf_file)

Sets appropriate permissions on log dir so that Router can run as a Windows service.

This function first obtains logging_folder (first it checks Router config file, if not found there, it uses the predefined default) and then sets RW access for that folder, such that Router can run as a Windows service

Parameters
conf_filePath to Router configuration file
Exceptions
std::runtime_erroron any error (i.e. opening/parsing config file fails, log dir is bogus, setting permissions on log dir fails)
Note
this function is private to this compilation unit, but outside of unnamed namespace so it can be unit-tested.
we do not care about the logfile access rights. The assumption is that if the Service creates the file it will have a proper rights to write to it. If the file already exists and is missing proper rights there will be an error - we are letting the user to fix that.
