Detailed Description

Class to decode or encode a stream of default value for instant table.

The decode/encode are necessary because that the default values would b kept as InnoDB format stream, which is in fact byte stream. However, to store them in the DD se_private_data, it requires text(char). So basically, the encode will change the byte stream into char stream, by spliting every byte into two chars, for example, 0xFF, would be splitted into 0x0F and 0x0F. So the final storage space would be double. For the decode, it's the converse process, combining two chars into one byte.
