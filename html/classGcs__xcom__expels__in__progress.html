Detailed Description

The set of expels we have issued but that have not yet taken effect.

An expel is identified as the pair (m, c), which means that we expelled the member m based on view(s) pertaining to the XCom configuration/membership c.

Whenever we issue the expel of some member m, we take note in this structure; see remember_expels_issued.

Whenever we process a view where some member m that we have expelled is no longer present, we remove m from this structure; see forget_expels_that_have_taken_effect.

Please note that we keep track of the configuration ID c where we expelled m to guard against the possibility of removing m from this strucutre due to receiving a view "from the past," e.g. receiving a view from configuration b < c where m is not a member, after having expelled m due to views from configuration c. It is unclear whether XCom can sometimes deliver views "from the past," so we account for this situation by only removing m from this structure if we receive a view from a configuration d > c where m is no longer present.
