377  /** @brief Creates Router account with compliant password
378  *
379  * Creates Router account with a (self-generated) password that will pass
380  * Server's password policy. It first tries creating a hashed password using
381  * mysql_native_password plugin. If that fails, it falls back to using
382  * plaintext password, which the Server may reject for not being strong
383  * enough. If that's the case, it will generate another password and try again
384  * 2 more times (for a total of 3 password-generation attempts), after which
385  * it will give up.
386  *
387  * @note This is a higher-level method, with smart logic that drives calls to
388  * lower-level create_accounts() method.
389  *
390  * @param user_options key/value map of bootstrap config options
391  * @param username Router account to be created - the username part
392  * @param hostnames Router accounts to be created - the hostname part
393  * @param password Password for the account
394  * @param password_change_ok password is allowed to be changed if needed
395  * @param if_not_exists if true, CREATE USER IF NOT EXISTS will be used
396  * instead of CREATE USER
397  *
398  * @returns password
399  *
400  * @throws std::logic_error on not connected
401  * std::runtime_error on bad password
402  * MySQLSession::Error on other (unexpected) SQL error
403  */
404  std::string create_accounts_with_compliant_password(
405  const std::map<std::string, std::string> &user_options,
406  const std::string &username, const std::set<std::string> &hostnames,
407  const std::string &password, bool password_change_ok, bool if_not_exists);
408
