53 /**
54  State_tracker
55  -------------
56  An abstract class that defines the interface for any of the server's
57  'session state change tracker'. A tracker, however, is a sub- class of
58  this class which takes care of tracking the change in value of a part-
59  icular session state type and thus defines various methods listed in this
60  interface. The change information is later serialized and transmitted to
61  the client through protocol's OK packet.
62 
63  Tracker system variables :-
64  A tracker is normally mapped to a system variable. So in order to enable,
65  disable or modify the sub-entities of a tracker, the user needs to modify
66  the respective system variable either through SET command or via command
67  line option. As required in system variable handling, this interface also
68  includes two functions to help in the verification of the supplied value
69  (ON_CHECK) and the updation (ON_UPDATE) of the tracker system variable,
70  namely - check() and update().
71 */
72

291  inline void update_change_flags(THD *thd) {
292  tx_changed &= ~TX_CHG_STATE;
293  // Flag state changes other than "is DML"
294  tx_changed |=
295  ((tx_curr_state & ~TX_STMT_DML) != (tx_reported_state & ~TX_STMT_DML))
